Sentiment Analysis of ChatGPT-related Tweets¶

Define the problem¶

This dataset of ChatGPT-related Tweets offers a snapshot of the public conversation around ChatGPT on Twitter. It comprises a CSV file with data on half a million tweets, tracking keywords, hashtags, mentions, and other relevant information. This dataset provides valuable training data for deep learning models aimed at understanding public perception and discourse around conversational AI. Researchers can leverage this data to train models for various tasks, including sentiment analysis to gauge public opinion, trend identification to uncover evolving topics, user engagement prediction to understand interaction drivers, and assessment of AI event impact on public discourse.

The input data from this dataset will be the 500k text of tweets, specifically the cleaned tweet content. We are trying to predict the sentiment of a tweet, which can be positive, neutral, or negative. This is a supervised classification problem, specifically a multi-class single-label classification task. It is multiclass as there are three sentiment categories in the dataset: positive, neutral, and negative. Each tweet can be classified into one of these three classes, so it's a multiclass problem. It is single-label as each tweet is assigned to one sentiment class only. For instance, a tweet cannot be both positive and neutral at the same time. This makes it a single-label classification, as each instance (tweet) has only one label (sentiment).

Choosing a measure of success¶

For sentiment analysis on tweets, where the dataset may be imbalanced (e.g., more neutral tweets than positive or negative ones), F1-score, precision, and recall are the most appropriate evaluation metrics. Precision measures how many of the tweets predicted as a certain sentiment (e.g., positive) are actually correct, making it important when false positives should be minimized. Recall quantifies how many actual tweets of a given sentiment were correctly identified, which is crucial when missing certain sentiments (false negatives) is costly. F1-score is the harmonic mean of precision and recall, providing a balanced measure that is particularly useful when both false positives and false negatives need to be minimized. Since sentiment classification involves multiple classes (positive, neutral, and negative), a macro-averaged or weighted F1-score ensures fair evaluation across all classes, even if the dataset is imbalanced. This approach avoids misleading results that could arise from using accuracy, and it is preferred over ROC AUC, which is more suited for binary classification.

Accuracy is not used because it can be misleading when dealing with imbalanced datasets. If the dataset contains significantly more neutral tweets than positive or negative ones, a model that simply predicts "neutral" most of the time could achieve high accuracy while failing to correctly classify positive or negative sentiments. This would give a false impression of good performance, even though the model is ineffective at distinguishing between classes. Also, ROC AUC focuses on ranking predictions rather than directly assessing the model’s ability to classify sentiments correctly. Since sentiment analysis typically involves positive, neutral, and negative classes, precision, recall, and F1-score provide more practical and interpretable insights into model performance.

Deciding on an evaluation protocol¶

For this dataset with 500K tweets, hold-out validation is preferred over K-Fold Cross-Validation and iterated K-Fold Cross-Validation due to its computational efficiency. Hold-out validation splits the data into training, validation, and test sets, and trains the model just once, which is much faster than K-Fold, where the model is trained K times. Given the large size of the dataset, K-Fold could be computationally expensive and time-consuming. Additionally, iterated K-Fold would further compound the issue by repeating K-Fold multiple times, making it infeasible for such a large dataset. Therefore, hold-out validation provides a good balance of model evaluation speed and reliability for this scenario, making it the most practical choice for efficient experimentation and testing.

In hold-out validation, the data is split 80% for training, 10% for validation, and 10% for testing. The model is trained on the training set, which contains the majority of the data. After training, the model’s performance is evaluated on the validation set, which acts as a proxy for unseen data, allowing for hyperparameter tuning and adjustments. Finally, once the model is fine-tuned using the validation set, its performance is assessed on the test set, which has never been used in the training process, to give an unbiased estimate of its generalization ability.

def clean_tweet(tweet):
    # Convert to lowercase
    tweet = tweet.lower()
    # Remove URLs
    tweet = re.sub(r'http\S+|www\S+|https\S+', '', tweet, flags=re.MULTILINE)
    # Remove user @ references and '#'
    tweet = re.sub(r'\@\w+|\#','', tweet)
    # Remove punctuation
    tweet = re.sub(r'[^\w\s]', '', tweet)
    # Tokenize the tweet
    tokens = word_tokenize(tweet)
    # Remove stop words
    tokens = [word for word in tokens if word.lower() not in stopwords.words('english')]
    # Lemmatize words
    lemmatizer = WordNetLemmatizer()
    tokens = [lemmatizer.lemmatize(word) for word in tokens]
    return ' '.join(tokens)

# Apply the cleaning function to the 'content' column
df['cleaned_content'] = df['content'].apply(clean_tweet)

def get_sentiment_score(tweet):
    sentiment_score = analyzer.polarity_scores(tweet)
    # Return the sentiment category
    if sentiment_score['compound'] >= 0.05:
        return 'positive'
    elif sentiment_score['compound'] <= -0.05:
        return 'negative'
    else:
        return 'neutral'

The dataset is preprocessed by mapping textual sentiment labels to numerical values, splitting the dataset into training, validation, and test sets, converting text into numerical features using TF-IDF, and computing class weights to handle imbalance.

The sentiment labels ('positive', 'neutral', 'negative') are mapped to numerical values (2, 1, 0) to make them compatible with machine learning models. The dataset is then split using a two-step approach: first, 80% of the data is used for training while 20% is set aside as a temporary set. This temporary set is further split equally into validation and test sets (10% each) to ensure a balanced distribution of sentiment classes.

Since we're restricted to Dense layers, we need fixed-length numerical input. TF-IDF works well for this. TF-IDF is applied to convert text data into numerical form by selecting the top 5000 most relevant words, ensuring that the transformation is consistently applied to the training, validation, and test sets. As the dataset is imbalanced, class weights are computed to assign higher importance to underrepresented classes, preventing the model from favoring majority classes.

Developing a model that does better than a baseline¶

To develop a small model that aims to outperform a simple baseline, we need to first establish what the baseline is, then build a simple model, and finally evaluate the performance.

Baseline model

The baseline model used in this context is a DummyClassifier that predicts the most frequent sentiment class in the dataset, which is commonly known as a majority class classifier. This model is implemented as a benchmark to evaluate the performance of more sophisticated models. It serves as a reference point, where the model simply predicts the most common sentiment (e.g., "positive") for all instances in the test set, regardless of the input data. By establishing this baseline, we can gauge whether a more complex model can outperform this simplistic approach, and how well it can capture meaningful patterns in the data.

Scaling up: developing a model that overfits¶

Building an overfit model is an essential step in understanding a model’s limitations. By intentionally training a model, often with one or two hidden layers, we can observe how it memorizes the training data instead of generalizing to unseen data. Monitoring both the training and validation loss helps us identify the point at which the model begins overfitting. This approach helps us evaluate the level of complexity required to capture patterns in the data and decide on the appropriate regularization techniques to mitigate overfitting.

Regularizing model and tuning hyperparameters¶

Regularization

It is identified that the model overfits at epoch 10. To address the issue of overfitting, regularization techniques like dropout and L2 regularization can be very effective. Dropout works by randomly deactivating a fraction of neurons during each training step, forcing the model to learn more robust features and preventing it from becoming overly reliant on specific neurons. This reduces the likelihood of the model memorizing the training data, thus improving its ability to generalize to unseen data. On the other hand, L2 regularization (also known as weight decay) adds a penalty term to the loss function that discourages large weights by penalizing the sum of squared weights. This helps prevent the model from fitting noise or outliers in the training data, making it more stable and less prone to overfitting. By combining these two regularization techniques, the model can achieve better generalization, as it learns to make more reliable predictions while avoiding the risk of overfitting to the training data.

    learning_rate = trial.suggest_float('learning_rate', 1e-5, 1e-2, log=True)

# Function to build deeper and wider model
def build_model(architecture, input_shape, output_units, dropout, l2_reg, learning_rate):
    model = Sequential()

    if architecture == "deeper":  # DEEPER MODEL (4 Dense Layers)
        model.add(Dense(256, activation="relu", kernel_regularizer=l2(l2_reg), input_shape=input_shape))
        model.add(Dropout(dropout))
        model.add(Dense(128, activation="relu", kernel_regularizer=l2(l2_reg)))
        model.add(Dropout(dropout))
        model.add(Dense(64, activation="relu", kernel_regularizer=l2(l2_reg)))
        model.add(Dropout(dropout))
        model.add(Dense(32, activation="relu", kernel_regularizer=l2(l2_reg)))
        model.add(Dropout(dropout))

    elif architecture == "wider":  # WIDER MODEL (More neurons per layer)
        model.add(Dense(256, activation="relu", kernel_regularizer=l2(l2_reg), input_shape=input_shape))
        model.add(Dropout(dropout))
        model.add(Dense(128, activation="relu", kernel_regularizer=l2(l2_reg)))
        model.add(Dropout(dropout))

    # Add last softmax layer
    model.add(Dense(output_units, activation="softmax"))

    model.compile(optimizer=RMSprop(learning_rate=learning_rate),
                  loss="categorical_crossentropy",
                  metrics=["Precision", "Recall"])

    return model
